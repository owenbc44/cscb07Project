Folder Descriptions:

Project Phases: are kept here for easy access by TA. 
Once a project phase deadline passes DO NOT CHANGE the contents of these folders.

Main: the Main branch where we will put together the code for the next upcoming phase.
Copy the contents of this folder into the next Project Phase folder when completed.
Only place code in here that is fully tested and debugged.

Tags: code placed here represents each phase of the project, as a history/ reference.
Once a phase passes push that phases' code into this folder.
Code placed here is NOT to be changed.

Tasks: the folder where we will divide the workload amongst ourselves.
Work ONLY in your OWN subfolder to avoid merge conflicts.

Agreement: kept here for easy access
